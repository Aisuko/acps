Q&A

Meta-review response

What changed in revision

ACPS supports both context-free and context-dependent reasoning via adaptive routing between standard and conditional front-door adjustment. Lightweight per-instance routing is paired with Sketch-of-Thought mediators built from diverse sampling and clustering, preserving reasoning diversity while reducing token usage.

  • Clarified efficiency accounting and causal estimation steps.
  • Explained how routing, mediator construction, and causal estimates interact.
  • Moderated novelty framing relative to prior causal prompting work.

Method

Adaptive Causal Prompting + Sketch-of-Thought

Instance-level routing

Select between standard and conditional front-door adjustments based on query context to preserve causal faithfulness.

Sketch-of-Thought mediators

Diverse sampling + clustering yields concise mediators that maintain reasoning coverage while reducing tokens.

Efficiency + robustness

Across GSM8K, HotpotQA, FEVER, Math, StrategyQA, MusiQue, and CommonsenseQA, ACPS improves accuracy and lowers inference cost.

Quickstart (notebooks)

  1. Open a Kaggle notebook (Python 3.11.13, GPU recommended).
  2. Install dependencies: pip install -r requirements.txt (the list is heavy; trim if you only need selected notebooks).
  3. Run acps.ipynb for the full pipeline, or the task-specific notebook under acps/.
  4. Use sots_datasets/ notebooks to regenerate Sketch-of-Thought mediators.
